# Data Generation Pipeline

## Overview
This repository contains the Data Generation Pipeline used as part of the Data Investigation System. Its main objective is to completely automate the process of collecting original remote sensing data, deriving useful machine learning features, training high-performance models, and finally predicting detailed land cover maps for recent or unsampled years based on those models.

The final output generated by this pipeline is a folder named `predictions`, which will then be consumed and displayed in the main application repository: [Data-Investigation-System](https://github.com/Zoorjz/Data-Investigation-System.git).

## Initial Data Setup
Before running the pipeline, you need the initial base foundational data, containing Sentinel-2 multi-spectral imagery and exact ESA WorldCover labels. You have two options for obtaining this initial data:

1. **Download Pre-packaged Data**: The easiest way is to download all the necessary original configured data directly from [this Google Drive link](https://drive.google.com/drive/folders/1At9ojXQe8cVU9RuzUWemnRE3cUCADcB2?usp=sharing). Extract it and ensure that both the `data` and `OriginalData` folders are placed directly in the root of this repository.
2. **Download from Source**: You can download the data completely manually and independently from the original sources (ESA website and Google Earth Engine). Please refer to the [Source Data Documentation](md/data_from_ESA_and_Sentinel_2_source.md) for detailed instructions on acquiring this data and how the exact folder structure must be configured.

## Project Structure & Pipeline Logic
The system goes through following sequentially ordered steps inside the scripts:
1. **Collect Original Data**: The pipeline begins by validating the existence of `OriginalData` (ESA WorldCover labels corresponding to 2020 and 2021) and downloads high-quality Sentinel-2 imagery for Nuremberg within precise time ranges (2017 to 2025).
2. **Derive Features**: Next, it produces localized training data. The software dynamically maps spatial features, applies noise-reduction filters, and builds image data patches. It converts the imagery (primarily `S2RGBNIR` composites) into structured Tabular Datasets.
3. **Train Models**: Using 2020 and 2021 images as labeled baseline ground truths, the pipeline automatically trains four standard Machine Learning architectures: Ridge Regression, Random Forest, Gradient Boosted Trees, and Multilayer Perceptron.
4. **Predict New Years**: Lastly, the pipeline acts as an inference engine. It applies the fully trained models across the full Sentinel-2 data spectrum for unseen or historical years, calculating the predicted land cover features locally.

## Running the Pipeline

Before running the pipeline, we highly recommend creating and activating a virtual environment. This prevents dependency conflicts with your system packages.

```bash
# Create a virtual environment
python -m venv .venv

# Activate the virtual environment
# On Windows:
.\.venv\Scripts\activate
# On macOS/Linux:
source .venv/bin/activate

# Install the required dependencies
pip install -r requirements.txt
```

Once the environment is configured and active, you can run the entire pipeline with a single command. By default, the script applies a smoothing denoising algorithm to the 2020 labels:

```bash
# To run the pipeline with the default denoising applied
python run_pipeline.py
```
or 
```bash
# To run the pipeline while preserving raw spatial noise
python run_pipeline.py --with_noise
```

## Output
Upon successful run, the core final product of this pipeline is a specifically named folder `predictions` containing inferred land-covers spanning different years and generated by multiple models. This name matches the strict naming requirements of the [Data Investigation System app](https://github.com/Zoorjz/Data-Investigation-System.git), allowing this folder to function as a direct copy-paste solution for mapping analysis and visualization.

## Model Training
Model training is implemented in the notebook `modeling.ipynb` which is inside the "Modeling_Pipeline/Model_1" folder in this repository. We trained 4 different models in this notebook.

### Training Tasks
The notebook trains models for two related prediction tasks:

1. **Task A -- Composition Prediction**
   - Predicts future land-cover composition percentages for each grid cell.
   - Main targets:
     - `Built-up %`
     - `Permanent water bodies %`
   - `Other %` is derived as the remaining percentage.

2. **Task B -- Change Prediction**
   - Predicts one-year land-cover change for each grid cell.
   - Main targets:
     - `delta Built-up %`
     - `delta Permanent water bodies %`

### Trained Models
The notebook trains four multi-output regression models:
- **Ridge Regression**
- **Random Forest**
- **Gradient Boosting**
- **Multilayer Perceptron (MLP)**

## Model Evaluation
Model evaluation is performed in `modeling.ipynb` and further analyzed in `analysis.ipynb` which lies inside the "Modeling_Pipeline/Model_1" folder.
- Hold-out stsrategies used are:
  - Temporal Hold-out for composition prediction
  - Spatial Block Cross-Validation for change prediction
- Standard Metrics used are:
  - MAE (Mean Absolute Error)
  - RMSE (Root Mean Square Error)
  - R2  (Coefficient of Determination)
- Change specific metrics used: False change rate, direction accuracy
- Stress tests: to measure robustness to noise (done for each model)
- Explainability and Trust:
  - Ridge coefficient analysis
  - Feature importance
  - Spatial error maps
  - Model disagreement maps
  - Explanation for meaningful and misleading examples

## Summary
This repository includes data collection, feature engineering, model training and evaluation, and generating detailed land covered predictions.
